This example shows how to send **Vercel AI SDK telemetry** to Strada. The app runs a small
agent loop with `gpt-5-mini`, three local tools, and OpenTelemetry spans for each model call
and tool call.

```bash
pnpm start
```

The package assumes `strada` and `sigillo` are installed globally. Secrets come from
**Sigillo**, not `.env` files.

## What this example sends

The AI SDK emits **OpenTelemetry spans** when `experimental_telemetry.isEnabled` is set.
Because `initStrada()` registers the global OpenTelemetry provider first, those spans flow
to the Strada collector automatically.

```txt
example-ai-sdk process

┌────────────────┐
│ initStrada()   │
└───────┬────────┘
        │ registers OTel providers + OTLP exporters
        ▼
┌───────────────────────────────────────────────────────────────┐
│ generateText({ experimental_telemetry: { isEnabled: true } }) │
└───────┬───────────────────────────────────────────────────────┘
        │ emits OTel spans
        ▼
┌──────────────────┐
│ Strada collector │
└───────┬──────────┘
        ├──────────▶ otel_traces
        └──────────▶ otel_logs
```

The span tree for this example looks like this:

```txt
ai.generateText                         full agent run
  ├─▶ ai.generateText.doGenerate         first OpenAI request
  ├─▶ ai.toolCall                        getProjectStatus
  ├─▶ ai.generateText.doGenerate         model sees project status
  ├─▶ ai.toolCall                        estimateFixCost
  ├─▶ ai.generateText.doGenerate         model sees estimate
  ├─▶ ai.toolCall                        createTodo
  └─▶ ai.generateText.doGenerate         final answer
```

Strada stores those spans in `otel_traces` with the normal OTel ClickHouse schema. The most
useful AI SDK attributes are inside `SpanAttributes`.

| Attribute | Example | Meaning |
| --- | --- | --- |
| `ai.operationId` | `ai.generateText` | AI SDK operation type |
| `ai.telemetry.functionId` | `example-ai-sdk-agent-loop` | Stable name for this call site |
| `ai.model.id` | `gpt-5-mini` | Requested model |
| `ai.model.provider` | `openai` | Provider package |
| `ai.prompt` | prompt text | High-level prompt, when recorded |
| `ai.prompt.messages` | JSON messages | Provider request messages |
| `ai.response.text` | final answer | Model output, when recorded |
| `ai.response.toolCalls` | JSON tool calls | Tool calls generated by the model |
| `ai.toolCall.name` | `createTodo` | Tool name for `ai.toolCall` spans |
| `ai.toolCall.args` | JSON args | Tool input |
| `ai.toolCall.result` | JSON result | Tool output |
| `gen_ai.request.model` | `gpt-5-mini` | Standard GenAI semantic convention |
| `gen_ai.usage.input_tokens` | `316` | Standard input token count |
| `gen_ai.usage.output_tokens` | `101` | Standard output token count |

The example also writes two custom log events to `otel_logs`:

```txt
ai_sdk_agent_started
ai_sdk_agent_finished
```

## Quick setup

Create or select a Strada project first:

```bash
strada projects list
strada projects create example-app
```

Store the secrets in Sigillo. `STRADA_PROJECT_ID` may already exist for this repo.
`strada projects create example-app` prints a server-side token once. If you need another token,
run `strada tokens create example-ai-sdk --scope ingest`.

```bash
sigillo setup
sigillo secrets set OPENAI_API_KEY "sk-..." -c dev
sigillo secrets set STRADA_PROJECT_ID "01K..." -c dev
sigillo secrets set STRADA_TOKEN "str_..." -c dev
```

Run the example:

```bash
pnpm start
```

Expected output:

```txt
{"stepNumber":0,"finishReason":"tool-calls","toolCalls":["getProjectStatus"]}
{"stepNumber":1,"finishReason":"tool-calls","toolCalls":["estimateFixCost"]}
{"stepNumber":2,"finishReason":"tool-calls","toolCalls":["createTodo"]}
{"stepNumber":3,"finishReason":"stop","toolCalls":[]}

Final answer:
...
```

## Minimal AI SDK instrumentation

The important part is initializing Strada before calling the AI SDK:

```ts
import { openai } from '@ai-sdk/openai'
import { generateText, stepCountIs, tool } from 'ai'
import { initStrada, flush, shutdown } from '@strada.sh/sdk'
import { z } from 'zod'

initStrada({
  projectId: process.env.STRADA_PROJECT_ID!,
  token: process.env.STRADA_TOKEN!,
  service: 'my-ai-agent',
})

const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Check project status and create one todo.',
  tools: {
    createTodo: tool({
      description: 'Create a todo.',
      inputSchema: z.object({ title: z.string() }),
      execute: async ({ title }) => ({ id: 'todo_1', title }),
    }),
  },
  stopWhen: stepCountIs(5),
  experimental_telemetry: {
    isEnabled: true,
    functionId: 'my-ai-agent-loop',
  },
})

console.log(result.text)
await flush()
await shutdown()
```

For sensitive prompts, disable input and output recording:

```ts
experimental_telemetry: {
  isEnabled: true,
  functionId: 'my-ai-agent-loop',
  recordInputs: false,
  recordOutputs: false,
}
```

## Query with the Strada CLI

List recent AI SDK spans:

```bash
strada query "
SELECT
  Timestamp,
  SpanName,
  SpanAttributes['ai.operationId'] AS operation,
  SpanAttributes['ai.telemetry.functionId'] AS function_id,
  SpanAttributes['ai.toolCall.name'] AS tool_name
FROM otel_traces
WHERE ServiceName = 'example-ai-sdk'
  AND Timestamp >= now() - INTERVAL 1 HOUR
ORDER BY Timestamp DESC
LIMIT 30
" -p example-app
```

Inspect tool inputs and outputs:

```bash
strada query "
SELECT
  Timestamp,
  SpanAttributes['ai.toolCall.name'] AS tool_name,
  SpanAttributes['ai.toolCall.args'] AS args,
  SpanAttributes['ai.toolCall.result'] AS result
FROM otel_traces
WHERE ServiceName = 'example-ai-sdk'
  AND SpanName = 'ai.toolCall'
  AND Timestamp >= now() - INTERVAL 1 HOUR
ORDER BY Timestamp ASC
LIMIT 20
" -p example-app
```

See token usage per model request:

```bash
strada query "
SELECT
  Timestamp,
  SpanAttributes['gen_ai.request.model'] AS model,
  SpanAttributes['gen_ai.usage.input_tokens'] AS input_tokens,
  SpanAttributes['gen_ai.usage.output_tokens'] AS output_tokens
FROM otel_traces
WHERE ServiceName = 'example-ai-sdk'
  AND SpanName = 'ai.generateText.doGenerate'
  AND Timestamp >= now() - INTERVAL 1 HOUR
ORDER BY Timestamp DESC
LIMIT 20
" -p example-app
```

Read the custom lifecycle logs:

```bash
strada query "
SELECT
  Timestamp,
  Body,
  LogAttributes['event.name'] AS event_name,
  LogAttributes['custom.model'] AS model,
  LogAttributes['custom.steps'] AS steps
FROM otel_logs
WHERE ServiceName = 'example-ai-sdk'
  AND Timestamp >= now() - INTERVAL 1 HOUR
ORDER BY Timestamp DESC
LIMIT 20
" -p example-app
```

## End-to-end flow

```txt
┌─────────────────┐
│ Sigillo secrets │
└───────┬─────────┘
        ├──────────▶ OPENAI_API_KEY
        ├──────────▶ STRADA_PROJECT_ID
        └──────────▶ STRADA_TOKEN
                   │
                   ▼
pnpm start ──▶ sigillo run ──▶ src/index.ts
                                   │
                                   ├──▶ initStrada()
                                   │      └──▶ OTLP HTTP/JSON exporters
                                   │
                                   └──▶ generateText()
                                          ├──▶ OpenAI responses API
                                          ├──▶ tool calls
                                          └──▶ AI SDK OTel spans
                                                      │
                                                      ▼
                                 {projectId}-ingest.strada.sh/v1/traces
                                                      │
                                                      ▼
                                         Tinybird / ClickHouse tables
                                                      │
                                                      ▼
                                             strada query "SELECT ..."
```

## Notes

- The model is hardcoded to **`gpt-5-mini`** so runs are reproducible.
- `maxRetries: 0` avoids burning quota on repeated failed model calls.
- The collector stores `TraceFlags` as the W3C trace flags byte. OTLP span flags can contain
  extra bits, so the collector masks them before writing to ClickHouse/Tinybird.
